Library Catalogue Manager

Babel

A self-hosted web app to manage your personal book collection. Track books, upload covers, mark items as borrowed or archived, and extract metadata from photos using OCR — all from your browser.

FastAPI SvelteKit SQLite Tesseract OCR Docker Python · TypeScript

Features

📚

Book Catalogue

Add, edit, and delete books with rich metadata: title, author, publisher, dates, language, and notes.

🖼️

Cover Images

Upload covers manually or auto-fetch from Open Library by title and author.

📷

OCR Scanning

Photograph a book's cover or title page and Tesseract will pre-fill the metadata form for you.

🔖

Borrowed & Archived

Dedicated views for lent-out books and books stored off-site, with one-click status changes.

🔍

Live Search

Debounced search across every section filters by title and author as you type.

📥

CSV Import

Bulk-import an existing spreadsheet catalogue in one command, with dry-run preview and duplicate detection.

Installation

Babel ships as a two-container Docker Compose app (frontend + backend). Docker is the recommended way to run it — no Python or Node.js setup required on the host.

Docker (recommended)

Prerequisites

  • Docker Engine 24+ or Docker Desktop
  • Docker Compose v2 (ships with Docker Desktop / modern Docker Engine)

Clone & run

git clone https://github.com/adelacalle/babel.git
cd babel
docker compose up --build

That's it. Docker pulls/builds the images, creates a SQLite database in ./data/db/, and starts both services.

ℹ️

On subsequent starts you can drop the --build flag: docker compose up.

ServiceURLDescription
Frontendlocalhost:3000SvelteKit web app
Backend APIlocalhost:8000FastAPI · JSON REST
API docslocalhost:8000/docsSwagger UI (interactive)

All data (SQLite database and uploaded cover images) persists in the ./data/ directory on your host machine and survives container restarts.

Development setup

Run each service directly on your machine for hot-reload during local development.

Prerequisites

  • uv — fast Python package manager
  • Node.js 20+
  • Tesseract OCR — sudo apt install tesseract-ocr on Ubuntu/Debian

Backend

cd backend
uv sync
uv run uvicorn app.main:app --reload --port 8000

Frontend

cd frontend
npm install
npm run dev

The Vite dev server runs on localhost:5173 and proxies API requests to the backend automatically. API docs remain at localhost:8000/docs.

Quickstart

Get from zero to a running catalogue in under two minutes.

1

Start the app

git clone https://github.com/adelacalle/babel.git && cd babel
docker compose up --build

Wait for the "Application startup complete" log line from the backend.

2

Open the app

Visit http://localhost:3000 in your browser. You'll land on the Catalogue page — empty for now.

3

Add your first book

Click + Add Book and fill in at least a title and author, then click Add Book. The card appears in the catalogue immediately.

4

Fetch a cover

Open the book's detail page and click Fetch Cover. Babel queries Open Library and downloads the cover image automatically.

5

Try OCR scanning

Click Scan in the sidebar, upload a photo of a book cover, and hit Scan 1 image. Review the pre-filled form and save.

✅

Tip: Photos of the title page or copyright page often yield the most complete metadata (title, author, publisher, year, ISBN).

Tutorial

A walk-through of every major feature in Babel.

1 · Adding a book manually

Click Catalogue in the sidebar, then + Add Book. Fill in at minimum Title and Author.

For dates you can enter just a year (1605), a year and month (2023-05), or a full date (2023-05-15). Use Original Publication for the first edition year and Publishing Date for the copy you own — handy for classics with many editions.

ℹ️

All fields except Title and Author are optional and can be edited later.

2 · Adding a cover image

From the book's detail page you have two options:

  • Fetch Cover — searches Open Library by title and author and downloads the cover automatically. Works best for well-known books with an ISBN.
  • Upload — click the Upload button to pick a local image (JPEG, PNG, or WebP).

3 · Scanning a book with OCR

This is the fastest way to add a book when you have a physical copy in hand.

1

Click Scan in the sidebar.

2

Drag and drop — or click to browse — one or more photos of the book. The best shots are:

  • Front cover — usually has the title and author.
  • Title page (inside the front cover) — often has publisher and year.
  • Copyright page (verso of the title page) — edition year, ISBN, original publication info.
3

Click Scan X image(s). Tesseract extracts the text and populates the Review & Save form.

4

Check each field, correct any mistakes, then click Save Book. Expand View raw OCR text if a field looks wrong — you can copy-paste from there.

✅

Tip: Good lighting matters more than resolution. Flat, evenly lit photos give much better OCR results than angled or shadowed shots.

4 · Borrowing and returning books

  • Open a book's detail page and click Mark Borrowed. The book moves to the Borrowed section.
  • To return it, open the detail page and click Return, or go to Borrowed and click the Return button on the card.
  • The book's status reverts to Available and it reappears in the main catalogue.

5 · Archiving books

Use Archived status for books stored outside the library (e.g., in boxes in a storage room).

  • Open a book's detail page and click Archive.
  • To restore it, go to Archived and click Restore on the card, or use the Restore button on the detail page.

6 · Searching

Every section (Catalogue, Borrowed, Archived) has a search bar at the top. Start typing — results filter by title and author in real time. The search is debounced so it waits until you pause before hitting the API.

7 · Importing from a CSV file

If you already have a catalogue in a spreadsheet, backend/import_csv.py can bulk-import it in one command. Always run a dry-run first to preview what will happen.

Step 1 — dry run

cd backend
uv run python import_csv.py /path/to/catalogue.csv --dry-run

Step 2 — import

uv run python import_csv.py /path/to/catalogue.csv

A summary is printed at the end:

==================================================
  Added:            312
  Skipped (dup):      4
  Skipped (empty):    1
  Errors:             0
==================================================
ℹ️

Running the import twice on the same file is safe — all rows will be detected as duplicates and skipped.

Running inside Docker

docker compose cp /path/to/catalogue.csv backend:/tmp/catalogue.csv
docker compose exec backend uv run python import_csv.py /tmp/catalogue.csv --dry-run
docker compose exec backend uv run python import_csv.py /tmp/catalogue.csv

CSV format reference

The import script expects the following column headers:

autor/a, título, título original, editorial, traductor/a, año publicacion, año edicion, idioma, etiquetas
CSV columnDB fieldNotes
autor/aAuthorRequired — rows without this are skipped
títuloTitleRequired — rows without this are skipped
título originalOriginal TitleOmitted if identical to title
editorialPublisher
año publicacionPublishing DateFree-text; year-only values like 1951 are fine
año edicionEdition Date
idiomaLanguage
traductor/aNotes (Translator)Combined into the Notes field
etiquetasNotes (Tags)Combined into the Notes field

All imported books are assigned status Available.

Project structure

babel/
├── backend/                 # FastAPI + SQLite
│   ├── app/
│   │   ├── main.py          # App entry point & middleware
│   │   ├── database.py      # SQLAlchemy setup
│   │   ├── models.py        # Book ORM model
│   │   ├── schemas.py       # Pydantic request/response schemas
│   │   ├── routers/         # API endpoints (books, borrowed, archived, ocr)
│   │   └── services/        # OCR + cover fetch services
│   ├── import_csv.py        # Bulk import script
│   └── pyproject.toml       # Python dependencies (uv)
├── frontend/                # SvelteKit (TypeScript)
│   └── src/
│       ├── routes/          # Pages: /, /books, /borrowed, /archived, /scan
│       └── lib/             # Components, API client, types
├── data/                    # Persisted data (git-ignored)
│   ├── db/                  # SQLite database file
│   └── uploads/covers/      # Uploaded & fetched cover images
└── docker-compose.yml
